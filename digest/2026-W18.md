## Emulation

### "Granny Smith" Browser-First Mac Emulator Receives Discussion at E-Maculation and 68kMLA

A hobby project called "Granny Smith," posted by user **PappaDF** in February 2026, continues to attract discussion on both E-Maculation and 68kMLA forums. 
The developer described it as a browser-first Macintosh emulator, available to try live at a GitHub Pages URL, and recommended Chromium-based browsers due to mouse movement issues in Safari.
 
The CPU instruction decoder is under 550 lines and all opcode implementations total less than 1,000 lines — not cycle-accurate, but described as accurate enough to pass the MacTest motherboard and floppy hardware tests without errors.
 
Features include background checkpointing in IndexedDB (allowing browser reloads without session loss), full checkpoint download/restore via drag-and-drop, and the ability to mount compressed disk images directly through drag-and-drop.
 The 68kMLA thread on this emulator showed a reply dated **April 11, 2026**.

**Source:** [E-Maculation Forum](https://www.emaculation.com/forum/viewtopic.php?p=81855) | [68kMLA Thread](https://68kmla.org/bb/threads/dingusppc-a-more-accurate-powerpc-mac-emulator.51537/)

### DingusPPC: Active Development Continues, No New Build This Week


DingusPPC is a Power Mac emulator for Windows, Linux, and macOS that aims to more accurately emulate actual hardware rather than patching the ROM and RAM as SheepShaver does. The most functional machines are the PowerMac 6100 and the PowerMac G3 Beige, though other machines are also implemented to varying stages of completion. The emulator is under active development and updated frequently.
 The E-Maculation DingusPPC forum board was last checked as of **April 21, 2026**, with no new posts recorded in the final week of April. The most recent macOS build dates to January 27, 2025 (ARM64) and July 24, 2024 (Intel). 
A WebAssembly port of DingusPPC is available and runs on Infinite Mac.


**Source:** [E-Maculation DingusPPC Forum](https://www.emaculation.com/forum/viewforum.php?f=38) | [E-Maculation DingusPPC Wiki](https://www.emaculation.com/doku.php/dingusppc)

### Infinite Mac: Browser Emulation Platform Remains Available Across Full Classic Mac Range


Infinite Mac is a collection of classic Macintosh and NeXT system releases and software accessible from a web browser, covering any version of System Software, Mac OS, Mac OS X, or NeXTStep from the 1980s, 1990s, or early 2000s. An "Infinite HD" disk with representative software from each era is also available.
 In March 2025, 
Infinite Mac gained the ability to run early Mac OS X, with 10.1 and 10.3 as the best-supported versions
, using PearPC as the underlying emulator. No new Infinite Mac releases were identified this week. 
Infinite Mac is backed by Mini vMac, Basilisk II, and SheepShaver depending on the OS selected, and supports importing or exporting BlueSCSI disk images.


**Source:** [Infinite Mac](https://infinitemac.org/)

### BlueSCSI Documentation Updated April 17, 2026


BlueSCSI is an open-source, open-hardware SCSI solution for vintage computers, designed to replace a SCSI hard drive inside a vintage computer or device.
 
The BlueSCSI wiki documentation was generated on 2026-04-17
, indicating an update to community-maintained documentation this week. 
With a Pico-W, BlueSCSI can emulate a DaynaPORT SCSI/Link, bringing wireless networking to vintage machines — enabling web browsing and FTP connections.
 
Initiator Mode allows connecting a vintage drive directly to the BlueSCSI to create a disk image on SD card, described as an archival tool.


**Source:** [BlueSCSI Documentation](https://bluescsi.com/docs/)

## Other

### Reverse-Engineering Classic Mac OS: elliotnunn cdg5 Project on GitHub


The `cdg5` project by GitHub user elliotnunn is a coordinated effort to reverse-engineer classic Mac OS, including patches to the "SuperMario" source to compile a System 7.1 file or Mac OS ROM (~9.2.2), thorough reversal of the Power Macintosh NanoKernel, tools to dump/patch/rebuild Mac OS ROM files, and utilities to get data into and out of HFS disk images and resource forks.
 No new commits were identified this week, but the project remains one of the more technically detailed open Mac OS reverse-engineering efforts publicly available.

**Source:** [GitHub — elliotnunn/cdg5](https://github.com/elliotnunn/cdg5)
