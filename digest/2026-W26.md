### cozyMIDI — New 68k MIDI File Player
A developer using the handle Mu0n posted a new application called **cozyMIDI** to the 68kMLA Software forum on June 20, 2026 (Friday). 
The app plays standard MIDI files assuming you have an external module plugged into the modem port, and loads and plays both Type 0 (single-track) and Type 1 (multi-track) standard MIDI files, keeping them in a pre-calculated form for efficient playback.
 
The code reconfigures the serial port and sends all data through it directly, without using the Apple MIDI driver, OMS, or QuickTime.
 The thread was still receiving replies as of this morning and had accumulated 14 responses within 24 hours.

**Source:** [68kMLA — cozyMIDI thread](https://68kmla.org/bb/threads/cozymidi-new-app-that-plays-std-midi-files-68k.52556/)

## App Revivals

*(Modern ports or revivals of classic Mac applications)*

### Lunacy — Lunatic Fringe on Modern macOS (Beta)
A developer using the handle **Halcyon** posted a thread to 68kMLA on June 22, 2026 (today, at 12:21 AM) announcing a new macOS application called **Lunacy**. 
The app emulates enough of the After Dark engine to run the original Lunatic Fringe After Dark module file, inspired by Greg Parker's earlier Fringe Player; as with Fringe Player, the user must supply the Lunatic Fringe module from an install of More After Dark or After Dark Deluxe.
 
The 1.0b1 download is available and the app runs full-screen with an optional CRT shader and a live global high-score board; it is built as a universal binary for macOS 13+.
 The author is soliciting feedback on notarization, framerate, and bugs. 
Lunatic Fringe is a fast-paced space shooter originally packaged as a module for After Dark on Macintosh System 7, programmed by Ben Haller.


**Source:** [68kMLA — Lunacy: Lunatic Fringe on modern macOS](https://68kmla.org/bb/threads/lunacy-lunatic-fringe-on-modern-macos.52575/)

### Lisa FPGA Project — Interview and Wider Coverage
A thread posted to 68kMLA on June 20, 2026 (Friday) links to a Classic Computing interview with computer engineering PhD student **Alex Anderson-McLeod** discussing the open-source **LisaFPGA** project. 
The Apple Lisa is one of the rarest, most expensive, and notoriously fragile machines in vintage computer history; Alex's project aims to change that by replacing complex, aging, and frequently battery-bombed circuit boards with a single, modern FPGA board.
 
The project achieves cycle-accurate performance to run original Apple Lisa software while introducing modern upgrades including HDMI video output, USB keyboard/mouse support, and onboard hard disk emulation, all open-source.
 
The custom board is centered around an Artix 7-100T FPGA, coupled to 2MB of SRAM with emulated hard drive, floppy, and serial interfaces.
 The project had already received significant attention when originally posted in early May; the interview thread this week brought it in front of the broader 68kMLA community.

**Source:** [68kMLA — Classic Computing interview: The Lisa FPGA Project](https://68kmla.org/bb/threads/classic-computing-interview-with-alex-anderson-mcleod-the-lisa-fpga-project.52549/)

### DingusPPC — Browser-Based PowerPC Emulator Progress
The Macintosh Repository's front page notes ongoing developments in browser-based PowerPC emulation. 
DingusPPC, described as a new emulator capable of running in WebAssembly, can boot Mac OS 9.2.2 and early Mac OS X releases in-browser; in parallel, a WebAssembly port of PearPC is also progressing, with both competing to offer faster PowerPC emulation in the browser.
 
As of early 2025, both emulators could boot Mac OS X 10.2 in a browser, though on a modern computer it takes nearly five minutes and runs too slowly for practical use.
 No new milestone update was found specifically this week, but the project remains in active development.

**Source:** [Macintosh Repository — front page notes](https://www.macintoshrepository.org/)
