### Infinite Mac — Status and Architecture


The Infinite Mac project uses Basilisk II, SheepShaver, Mini vMac, DingusPPC, Previous, and PearPC as its underlying emulator backends, hosted in separate repositories and included via Git submodules.
 
The project allows users to pick any version of System Software, Mac OS, Mac OS X, or NeXTStep from the 1980s through the early 2000s and run it within a virtual machine, with an "Infinite HD" disk containing representative era software also available. Custom machine and disk configurations can be embedded into external sites.
 No new Infinite Mac releases were detected this week, though the project remains actively used as a reference and discussion topic across community forums.

**Source:** [Infinite Mac](https://infinitemac.org/) | [GitHub — mihaip/infinite-mac](https://github.com/mihaip/infinite-mac)

### DingusPPC and PearPC in WebAssembly — Background


Since late 2023, work has been ongoing on DingusPPC, an emulator running in WebAssembly capable of booting Mac OS 9.2.2 and early Mac OS X releases. In parallel, PearPC has also been ported to WebAssembly; as of early 2025, both can boot Mac OS X 10.2 in-browser, though at practical speeds that remain too slow for regular use.
 The Macintosh Repository continues to surface these options for users who want browser-based PowerPC emulation.

**Source:** [Macintosh Repository](https://www.macintoshrepository.org/)

## Software

### MacSurf v1.4 "Open House" — Native Browser for Mac OS 9

The most active project visible this week in the classic Mac software space is MacSurf, a native web browser for Mac OS 9 PowerPC. 
The latest release, MacSurf v1.4 "Open House" (dated 2026-06-01), closed 23 JavaScript-bridge issues in a single release, adding support for `setTimeout`/`setInterval`/`requestAnimationFrame`, `window.location`, `window.history`, `URL`, `URLSearchParams`, `classList`, `localStorage`, `fetch`, `MutationObserver`, `DOMParser`, `FormData`, and load/`DOMContentLoaded` events. The new probe suite scored JS 19/19 pass on a G3 iMac.



MacSurf is a native browser, built with CodeWarrior, the Carbon API, QuickDraw, and Open Transport — not a screenshot proxy or remote terminal trick — and targets the 64 MB memory floor of a 1999 iMac.
 
Since late May 2026 it speaks TLS 1.2 directly to the modern web through macTLS, a BearSSL-based stack that ships inside the browser binary with 121 trust anchors from the Mozilla CA bundle, requiring no proxy. The developers describe it as the first serious NetSurf port to Classic Mac OS, and the first browser shipped on Mac OS 9 with native CSS Grid, CSS custom properties, and ES5 JavaScript.



As of the week of June 29, the developer posted on the 68kMLA front page that MacSurf 1.5 is planned for release "this weekend," with the stated goal of better support for the 68kMLA forum itself so users can browse and post from OS 9. Some of the features in MacSurf are described as not present in the upstream NetSurf build.


**Source:** [GitHub — mplsllc/macsurf](https://github.com/mplsllc/macsurf) | [68kMLA Thread](https://68kmla.org/bb/threads/macsurf-a-real-netsurf-based-browser-for-mac-os-9-css3-es5-js-native-https.52332/) | [Hackaday Coverage](https://hackaday.com/2026/05/29/a-modern-web-browser-for-classic-mac-os/)

## Hardware

### TashTalk USB — New Board for LocalTalk-to-USB Bridging


A new USB-to-TashTalk board based on a CP2102N and a full-duplex transceiver has been developed, derived from Tashtari's Pi Hat. The goal is to make it straightforward to transfer files between any classic Mac with a LocalTalk port and a modern Mac, Linux, or Windows machine.
 
It uses unmodified TashTalk firmware and should work with existing TashTalk software. Schematics and board files are available on GitHub at the FeralFirmware/TailTalk repository.



A GUI for controlling the server is described as nearly ready for trials on Linux and Mac, with Windows support to follow.
 
The board can serve AFP 1.0–2.1 compatible file shares via EtherTalk or LocalTalk. The first run of 10 boards had a minor wiring error requiring bodge fixes before distribution to testers.



The TashTalk USB thread on 68kMLA showed activity as recently as June 28, 2026.


**Source:** [68kMLA — TashTalk USB Thread](https://68kmla.org/bb/threads/tashtalk-usb.52115/) | [GitHub — FeralFirmware/TailTalk](https://github.com/FeralFirmware/TailTalk/tree/main/TashTalkUSB)
